# Protocol Gap Tracker - Phase 2

## מטרת המסמך

מסמך זה הוא מסמך המעקב הראשי לשלב הבא של הפרויקט, לאחר סגירת פערי הליבה הראשונים ב-[protocol-gap-tracker.md](/Users/evyatarhazan/Desktop/project/emergency-protocol-diagram/docs/tracking/protocol-gap-tracker.md).

הוא נועד לקחת את התובנות מהשלב הקודם, ולהפוך אותן לרשימת פערים חדשה עד להשלמה מלאה של:

1. הוכחת כיסוי קליני מלאה מול מקור האמת
2. חוויית למידה ודיון משותף סביב כל צומת
3. שכבת מוצר שעובדת לא רק כפרוטוקול, אלא גם כסביבת למידה וקהילה קטנה

## מה כבר הושלם בשלב הקודם

- נסגרו פערי הזרימה הראשית והמסגור המוצרי הראשוני.
- נסגרו פערי המיפוי מול מקור האמת ליחידות המרכזיות.
- נוספו מקורות גלויים לצמתים המרכזיים.
- נסגרו פערי פיצול הצמתים העמוסים (`GAP-011`).
- הוכח ש-`push` ל-`main` נפרס אוטומטית לפרודקשן ב-Cloudflare Pages.

## תובנות שעוברות בירושה מהמסמך הקודם

1. מקור האמת נשאר תיקיית `איחוד הצלה` ב-Google Drive והמסמכים שנקלטו לריפו.
2. הזרימה הראשית נשארת קשיחה:
   `S -> התרשמות זירה -> AVPU -> המשך לפי ממצאים`
3. שכבות משניות מותרות רק אם הן תומכות בזרימה הראשית ולא מחליפות אותה.
4. שכבת הדיון/קהילה קיימת טכנית, אבל עדיין לא מוכחת כמוצר עובד, שימושי ומובנה.
5. יש עדיין פער בין "כיסוי נראה טוב" לבין "כיסוי מוכח, מדוד ומלא".

## סטטוסי עבודה

- `open` - טרם התחיל
- `in_progress` - בעבודה
- `blocked` - תקוע ותלוי בגורם אחר
- `done` - הושלם ואומת

## רמות תיעדוף

- `P0` - קריטי להשלמה מלאה של החזון

- `P2` - שיפור משמעותי, אך לא החסם הראשון
