# REM-001A BLS Scope Audit

## Scope

סבב audit ייעודי ל-`REM-001A`, ממוקד בצמתי הליבה שנבדקו עד כה:

`scene -> AVPU -> A -> B -> C -> D -> E -> secondary survey -> head_trauma -> unconscious_breathing -> seizure -> syncope -> poisoning_overdose -> meningitis -> septic_shock -> stroke`

## Goal

לוודא שה-flow הראשי המיועד ל:

- `BLS`
- `חובש`
- `נהג אמבולנס`

אינו מציג טיפול `ALS` או פרוצדורה מתקדמת כאילו היא חלק מברירת המחדל של המטפל בשטח.

## Findings

### What was checked

1. האם ההנחיה הראשית היא זיהוי, תמיכה בסיסית, ניטור, הזעקת ALS ופינוי.
2. האם טיפול מתקדם מוצג כפעולה שהחובש מבצע בעצמו, או רק כאפשרות `אם זמין / אם הפרוטוקול המקומי וההסמכה מאפשרים`.
3. האם יש מסר שעלול לבלבל תלמיד לחשוב שפרוצדורת ALS היא חלק מה-flow הראשי.

### Result

- לא נמצא בצמתי הליבה שנבדקו טיפול `ALS` שמוצג כברירת מחדל לחובש/נהג אמבולנס.
- במקומות שבהם מופיעים:
  - `נלוקסון`
  - `ניטרוגליצרין`
  - `ECG / מוניטור`
  - `EpiPen / אפינפרין`

  הניסוח נשאר מותנה ב:
  - `לפי פרוטוקול מקומי`
  - `אם זמין`
  - `אם הצוות מוסמך`
  - או כפעולת escalation להזעקת `ALS`

- לכן שכבת הליבה שנבדקה תואמת עקרונית ל-BLS ואינה דורשת תיקון קוד נוסף כרגע לצורך `REM-001A`.

## Closure

בהינתן הסקופ שנבדק, `REM-001A` נחשב סגור.
