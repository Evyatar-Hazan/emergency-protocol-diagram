# Protocol Gap Tracker

## מטרת המסמך

מסמך זה הוא מסמך המעקב הראשי לפערים בין:

1. המוצר הקיים באתר
2. מקור האמת מתוך תיקיית `איחוד הצלה` ב-Google Drive
3. חזון המוצר כפרוטוקול צעד-אחר-צעד מבוסס סיטואציה

המסמך נועד לשמש אותנו כבסיס עבודה שוטף עד להשלמת ההתאמה המלאה.

## עקרונות עבודה

- הזרימה הראשית של המוצר נשארת:
  `S -> התרשמות זירה -> AVPU -> המשך לפי ממצאים`
- מקור האמת הקליני והתפעולי הוא חומרי הדרייב.
- התפריט המשני בעתיד יאפשר קפיצה לסכמה ספציפית, בלי לשנות את קשיחות הזרימה הראשית.
- כל פער חייב לכלול:
  - הגדרה ברורה
  - דרך תיקון
  - סטטוס
  - קריטריון סגירה
  - דרך אימות

## סטטוסי עבודה

- `open` - טרם התחיל
- `in_progress` - בעבודה
- `blocked` - תקוע ותלוי בגורם אחר
- `done` - הושלם ואומת

## רמות תיעדוף

- `P0` - קריטי לחזון המוצר ולנכונות הקלינית
- `P1` - חשוב מאוד, אך לא חוסם לבדו את כל המערכת
- `P2` - שיפור משמעותי, אך משני לעומת פערי הליבה

## Definition of Done

פער נחשב סגור רק אם כל התנאים הבאים מתקיימים:

1. התוכן או המבנה עודכנו בפועל בקוד או במסמכי המקור.
2. בוצעה בדיקה מול מקור האמת הרלוונטי בדרייב.
3. בוצעה בדיקה ידנית ב-UI לצומת/מסלול הרלוונטי.
4. אם הייתה השפעה על הקוד:
   - `npm run build`
   - `npm test -- --runInBand`
5. אם הייתה דחיפה לענן:
   - אימות דיפלוי
   - אימות פרודקשן
6. הסטטוס במסמך הזה עודכן.
